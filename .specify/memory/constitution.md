# TradingClaw Constitution

## Core Principles

### I. Real-Time Data Integrity

Ingestion layers MUST handle data streams persistently and continuously. Rate limitations on exchanges should be managed gracefully using queues (e.g. BullMQ). Sequence gaps MUST be detected and filled via snapshots. A trading agent relies entirely on consistent tick flow; broken data means broken decisions.  
_Rationale: Crypto exchanges impose aggressive rate restrictions while simultaneously pushing critical WebSocket updates. An agent failing to track state effectively is guaranteed to misallocate capital._

### II. Quantitative Rigor

All LLM reasoning MUST be grounded in robust mathematical and statistical features computed locally. LLMs describe "why," but the math describes "what." Agent tool outputs MUST provide technical indicators, order flow metrics (VWAP, OB imbalances), and Kelly fraction numbers before qualitative reasoning occurs.  
_Rationale: Language models are prone to hallucination if asked to guess trend signals from raw numbers. Providing calculated Quant features minimizes the probability of erroneous assessments._

### III. Fail-Safe Risk Management (NON-NEGOTIABLE)

Circuit breaking heuristics MUST ALWAYS supersede agent commands. If Max Drawdowns (Daily and Peak-to-Trough) breach hard thresholds, the execution layer MUST forcefully close positions and halt new entries, ignoring the active Strategy or Agent Intent. Kelly fractional caps MUST never be exceeded.  
_Rationale: Agentic actions can loop or suffer logic collapse. Quantitative guardrails ensure the project's capital remains safe under adversarial or bug-ridden conditions._

### IV. Transparent Decision Journaling

Every inference generated by the Trading Agent—including the decision to HOLD—MUST be journaled completely. The log must capture the JSON structured decision, reasoning, confidence, state of the Quant signals, sentiment fed into it, and macro conditions.  
_Rationale: Without a transparent ledger of "what the agent saw and what it thought" during every tick, building fine-tuning datasets and RAG pipelines is impossible._

### V. Shared Multi-Agent Context

Memory and Context MUST be shared intelligently across specialized domains rather than monolithic system prompts. Context tracking and shared memory directories (influenced by `marcel/` architectures) MUST be used to isolate Ingestion, Quantitative Analysis, Sentiment classification, and Order Execution workloads out of the main Trader context where possible.  
_Rationale: Context windows are limited. By isolating context specific to various layers, we save token bandwidth for actual trade evaluation and hypothesis formulation._

## Technical Constraints & Standards

- **Typesafety**: Interfaces MUST be strongly typed (TypeScript). Event schemas across external integrations (WebSocket payloads to database writes) should employ validators (e.g. Zod) at ingestion bounds.
- **Data Persistence**: Heavy reliance on TimescaleDB for structured OHLCV/time-series data, combined with `pgvector` for embedding storage.

## Development Workflow

- Changes impacting `packages/trading/quant` or Risk constraints MUST require an isolated unit test validating edge cases and zero-division fallbacks.
- New strategies must ALWAYS cycle: `hypothesis` → `backtesting` → `testnet` → `live`, enforced systematically. There are no direct pushes to live strategy execution.

## Governance

Amendments to this constitution require standard PR documentation, discussion describing why constraints were altered, and explicit migration plans if testing/risk gates are relaxed.

- All Pull Requests must verify compliance against principles (Risk, Reporting).
- Versioning follows SemVer based strictly on changes to these Governance and Principle guidelines.
